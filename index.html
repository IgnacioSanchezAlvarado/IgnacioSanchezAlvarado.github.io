Summary: This implementation guide provides an overview of the Industrial AI Buddy, a digital twin powered by Amazon Bedrock solution. It includes the reference architecture, components, deployment steps, and configuration instructions for deploying the solution on Amazon Web Services (AWS). This guide is intended for Solution Architects, Business Decision Makers, DevOps Engineers, and Cloud Professionals who want to implement an interactive digital twin with AI capabilities in their environment.

Overview

Industrial AI Buddy is a cloud-rendered digital twin powered by an Amazon Bedrock AI assistant. This Unreal Engine-powered environment connects to an Amazon Bedrock agent, providing real-time information about a plant's technical details and processes. Users interact via voice with the AI guide, exploring the facility and learning about its operations. The system uses a Retrieval Augmented Generation (RAG) approach, incorporating plant documentation for comprehensive information, and an AI agent that can retrieve operational plant parameters.

Key Features

  • Interactive virtual 3D environment of a manufacturing facility
  • Amazon Bedrock voice assistant integration for natural language interaction
  • Documentation helper for plant processes and worker training scenarios
  • Real-time information retrieval about facility operations and equipment
  • Multilingual support (English, French, Spanish)

Business Impact

This solution is based on a real-world implementation by a major automotive customer. It provides an anonymized demo to engage with customers from manufacturing and energy industries, offering a powerful way to visualize and interact with complex industrial environments.

Architecture Overview

The Industrial AI Buddy solution combines a 3D digital twin environment with powerful AI capabilities from Amazon Bedrock. The architecture enables seamless voice interaction, knowledge retrieval, and visual representation of industrial facilities.

Technical Flow

  1. User connects directly to the Unreal Engine 3D application through Amazon GameLift Streams. They can navigate the environment to explore the facility using controller-based movement or WASD keys.
  2. When the user presses a button to invoke the GenAI assistant, they communicate via voice using an external microphone. A Lambda function receives the user prompt and user location data (which station they are near, nearby equipment, etc.). Amazon Transcribe converts voice to text.
  3. The Lambda function calls Amazon Bedrock with the following components:
    • Knowledge base query using the user prompt and environment metadata. The knowledge base contains facility and machinery equipment documentation.
    • Agent query using three tools: Knowledge base, Garnet framework tool (for contextual AI), and reparation tickets tool.
  4. The Bedrock text response is sent to Amazon Polly for voice conversion. The user receives voice and text responses to their questions (e.g., "What is this machine doing?", "Which processes take place in this station?", "How should I operate this equipment?", "Can you open a reparation ticket?", "What are the coordinates of this machine?").

AWS Services Used

AWS Service Role Description
Amazon Bedrock Core Provides the foundation model for AI assistant capabilities and knowledge base integration
Amazon GameLift Streams Core Delivers cloud-rendered 3D application to users' devices
Amazon API Gateway Core Manages API endpoints for communication between components
AWS Lambda Core Processes user inputs, queries knowledge base, and handles responses
Amazon S3 Core Stores documentation for the knowledge base
Amazon Transcribe Core Converts user voice input to text
Amazon Polly Core Converts text responses to natural-sounding voice
Amazon CloudFront Optional Delivers web application content for GameLift Streams integration

Deployment Guide

Deploying the Industrial AI Buddy solution involves three main steps:

Step 1: Deploy the Backend Bedrock Voice Interaction CDK Stack

The backend stack provides the AI capabilities, knowledge base, and API endpoints for the solution.

Prerequisites

  • AWS Account with appropriate permissions
  • AWS CLI installed and configured
  • Node.js and npm installed
  • AWS CDK installed (npm install -g aws-cdk)

Deployment Steps

Terminal
# Clone the repository
git clone https://github.com/IgnacioSanchezAlvarado/IgnacioSanchezAlvarado.github.io.git
cd IgnacioSanchezAlvarado.github.io/BedrockKnowledgeBaseVoice

# Install dependencies
npm install

# Deploy the CDK stack
cdk deploy

Setting Up the Knowledge Base

After deploying the CDK stack, you need to upload documentation to the S3 bucket and sync it with the knowledge base:

  1. Locate the S3 bucket name from the CDK output (DocumentBucketName)
  2. Upload your facility documentation to the S3 bucket:
    Terminal
    aws s3 cp facility_overview.md s3://your-document-bucket-name/
  3. Sync the knowledge base with the new documents:
    Terminal
    aws bedrock-agent start-ingestion-job \
    --knowledge-base-id your-knowledge-base-id \
    --data-source-id your-data-source-id

You can download a sample facility documentation file to add to your knowledge base:

Download Sample Facility Documentation

Step 2: Deploy the Solution in Amazon GameLift Streams

Amazon GameLift Streams allows you to stream the Unreal Engine application to users' devices without requiring them to install any software.

Creating an Application in GameLift Streams

  1. Navigate to the Amazon GameLift console
  2. Select "Streams" from the navigation pane
  3. Click "Create application"
  4. Enter a name for your application
  5. Configure the application settings according to your requirements
  6. Click "Create"

For detailed instructions on creating applications in GameLift Streams, refer to the Amazon GameLift Streams documentation.

Creating Stream Groups

  1. In the GameLift console, select your application
  2. Navigate to the "Stream groups" tab
  3. Click "Create stream group"
  4. Configure the stream group settings
  5. Click "Create"

For detailed instructions on creating stream groups, refer to the Amazon GameLift Stream Groups documentation.

Adding the Unreal Engine Application

You have two options for adding the Unreal Engine application to GameLift Streams:

Option 1: Use the Sample Game

You can use a pre-built sample game to test the solution:

Option 2: Use the Source Project

If you want to customize the Unreal Engine project:

  1. Clone the source repository: https://github.com/IgnacioSanchezAlvarado/IndustrialAIBuddyUEproject
  2. Open the project in Unreal Engine
  3. Replace the API Gateway URL in the code with your own
  4. Build the game
  5. Upload the built game to GameLift Streams

Step 3: (Optional) Deploy the Web Application

If you want to publish the application and share it with others, you can deploy a web application that connects with the GameLift Streams service:

Demo Guide

This section provides guidance on how to demonstrate the Industrial AI Buddy solution.

Demo Controls

Action Controller Keyboard/Mouse
Move character Left joystick WASD keys
Move camera Right joystick Mouse
Change language Press X Press L key
Ask question Press and hold B Press and hold Enter key
Reset app Press Select for 2 seconds Press Delete key for 2 seconds

Example Demo Flow (3-5 minutes)

  1. Walk to one of the closer areas (storage or working station) and explore the facility
    • While moving around, discuss the digital twin concept, business benefits, and use cases (training in safe environments, real-time collaborations, operations and maintenance)
  2. Approach one of the items/equipment. When close to it, press the AI assistant button (keep it pressed) and speak to the microphone:
    • Example question: "Can you explain what this machine is?"
    • Example question: "What are its operating parameters?"
    • Example question: "What are the recommended maintenance procedures?"
  3. Move to a different area (recommended: power generation or assembly area)
    • While moving, discuss the GenAI benefits applied to an industrial scope, how technical information can reach users in a conversational, immersive style
  4. Ask more questions:
    • Example question: "What are the processes taking place in this area?"
    • Example question: "What are their safety indications?"
    • Example question: "Can you elaborate more on the last guidance?"
  5. Finish demo and restart application
    • Discuss the technologies used (GameLift Streams, Bedrock, Unreal Engine)
    • Suggest ideas for building a similar PoC
  6. Optional: Show multilingual capability by pressing the language button and asking a question in a different language (available: English, French, Spanish)

Important Notes

  • Most users will struggle at the beginning with controls. Consider demonstrating a first question example yourself before passing control to them.
  • Explain to users that they need to keep the speak button pressed for the entire question duration.
  • Wait until the model has replied before asking new questions.
  • IMPORTANT: The app has a crash issue because the audio buffer is not properly refreshed and eventually overloads. To avoid this, reset the app every 3-4 questions (keeping Select/Delete buttons pressed for 2 seconds). If it crashes, reload the stream/executable.

Frequently Asked Questions

How much does this solution cost? What's included?

The main cost drivers are GameLift Streams and Amazon Bedrock. Both are usage-based services: stream time for GameLift and number of tokens for Bedrock. For detailed pricing information, visit GameLift Streams Pricing and Bedrock Pricing.

Is this solution available in my region? What are the deployment options?

Bedrock is available in multiple regions. GameLift Streams in EU is only available in Frankfurt and Ireland. In APJ, it's available in Tokyo. Check the AWS documentation for other regions.

Is there a trial/POC version available? Is it open source?

This is a demo to show what's possible. If interested, reach out to the AWS Spatial Computing team (through your account team), and they can guide you to build a solution for your business.

What is the scalability and performance capacity?

This is a fully serverless solution with high scalability. For GameLift Streams, you can increase stream groups. For Bedrock, you can increase quotas or use Provisioned Throughput for high utilization use cases.

What are the integration capabilities and technical requirements?

For streaming deployment, you only need a web browser and stable internet connection (minimum 10 Mbps). If the connection is not good, a local deployment can also be set up.

What is the typical implementation timeline and process?

Just a few weeks for a first PoC. For a production workload, a Digital Twin is a complex solution that usually takes several months to be created.

How does this compare to similar solutions in the market?

Most digital twin solutions don't include GenAI capabilities + a managed streaming service.

Troubleshooting

Common Issues

  • If the application crashes, restart it by reloading the stream/executable.
  • If voice recognition is not working, check your microphone settings and ensure you're keeping the speak button pressed for the entire duration of your question.
  • If the AI assistant doesn't respond, make sure your knowledge base is properly set up and synced with your documentation.
  • For connectivity issues with GameLift Streams, ensure you have a stable internet connection with at least 10 Mbps bandwidth.

Uninstall Instructions

To remove the deployed resources:

  1. Delete the GameLift Streams application and stream groups from the GameLift console.
  2. Delete the CDK stack:
    Terminal
    cd BedrockKnowledgeBaseVoice
cdk destroy
  3. If you deployed the web application, delete its CDK stack as well.
  4. Empty and delete any S3 buckets created by the solution.
